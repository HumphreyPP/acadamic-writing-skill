# Literature Review Writing

## Overview

Use this skill to turn source papers into review prose that reads evidence-led rather than citation-stacked. Build claims from the literature first, then write paragraphs that move from what kind of relationship exists to what direction the evidence points in.

## Core Workflow

### 1. Build an evidence map before drafting

Identify the claim units that the paragraph needs to carry before writing full prose.

For each claim unit, note:

- the relationship or type conclusion
- the directional finding, if one is supported
- boundary conditions, mechanisms, or exceptions
- the one or two sources that directly support that claim

Do not draft from a loose pile of references. If a source does not support a specific clause, do not attach it there as padding.

### 2. Draft with sentence-level support

Treat citation placement as part of the argument, not decoration.

Follow these rules:

- Give each sentence one main claim whenever possible.
- If a sentence contains more than one substantive claim, split it into clauses or separate sentences and support each unit directly.
- Keep each parenthetical citation lean. Do not place more than two references in a single parenthetical citation.
- If more than two references seem necessary, rewrite the prose so that different clauses carry different citations.
- Avoid ending a long sentence with a catch-all parenthetical that appears to support everything before it.
- Prefer placing citations immediately after the clause they support when that improves clarity.

Default target: each sentence, or each substantive clause inside a sentence, should have visible support.

## Paragraph Logic

### 3. Order claims from relation to direction

After any opening orienting sentence, structure body paragraphs in this order:

1. State the relationship, association type, distinction, or comparison first.
2. State the directional finding next.
3. Add boundary conditions, moderators, mechanisms, or disagreements after that.

Use this sequence unless the literature genuinely does not support a directional statement.

Examples of acceptable movement:

- first establish that a variable pair is associated, then specify whether the association is positive, negative, nonlinear, or mixed
- first establish that a factor matters, then specify whether it increases, reduces, strengthens, weakens, or redirects the outcome
- first establish that studies differentiate two mechanisms, then specify which mechanism dominates under which condition

Avoid stopping at empty formulations such as "`A` affects `B`" when the literature supports a more concrete statement.

If the evidence does not justify a directional claim, say so plainly instead of inventing certainty.

## Style Controls

### 4. Remove AI-sounding prose before finalizing

Run a deliberate prose pass after the evidence structure is in place.

Revise away these patterns:

- several consecutive sentences with the same length or syntax
- repeated sentence openings such as "in addition", "meanwhile", "notably", or "studies show"
- stacked short noun phrases that read like outline bullets turned into prose
- heavy use of quotation marks, colons, semicolons, and dashes for emphasis
- vague summary language that could fit almost any topic
- generic verbs such as "has an impact on" when the literature supports a more specific verb

Prefer this style:

- alternate shorter and longer sentences
- use plain academic prose instead of rhetorical signposting
- name the concrete pattern shown in the studies
- let citations carry authority instead of intensifiers

## Revision Pass

### 5. Run the final checklist

Before returning a draft, check the points in [revision-checklist.md](./references/revision-checklist.md).

At minimum, verify:

- no parenthetical citation contains more than two references
- no sentence hides multiple unsupported claims
- each body paragraph moves from relationship/type to directional finding
- abstract claims have been replaced with concrete findings where the literature allows
- obvious AI-writing traces have been revised out

## Output Preference

When writing from source literature, silently do the evidence mapping first even if the user asks directly for prose. Return polished prose unless the user explicitly asks to also see the outline, evidence table, or claim map.
