# Data Analysis Writing

## Overview

Use this skill to translate analytic output into precise prose. Separate what the data show from what the finding may mean, and make the directional pattern explicit whenever the evidence supports it.

## Core Workflow

### 1. Extract the claim sequence from the analysis

Before drafting, identify:

- the result or pattern that is directly observed
- the direction, magnitude, or contrast that matters
- the interpretation that can be justified
- the caveat, robustness point, or limit that should remain visible

### 2. Write results in a controlled order

Default paragraph sequence:

1. name the pattern or comparison
2. state the direction or contrast
3. add the statistic, coefficient, coded theme, or concrete evidence if relevant
4. interpret cautiously
5. note limits or robustness only where they matter

Do not blur description and interpretation into one vague sentence.

### 3. Avoid overclaiming

- Do not write causal language unless the design supports it.
- Do not imply theoretical contribution when only an empirical pattern has been shown.
- Do not generalize beyond the analyzed sample or context without support.

## Style Controls

- Prefer exact verbs such as "increases", "declines", "clusters", "diverges", or "co-occurs" over generic verbs.
- Keep numbers selective and meaningful instead of flooding the paragraph.
- Use [anti-ai-prose-checklist.md](../../shared/references/anti-ai-prose-checklist.md) to smooth the prose.
- Use [citation-discipline.md](../../shared/references/citation-discipline.md) when the analysis paragraph also discusses prior studies.

## Output Preference

When writing from tables, models, coded outputs, or summary notes, first extract the result, direction, interpretation, and caveat silently. Return prose by default unless the user explicitly asks to also see a result map, bullet outline, or table-to-text conversion notes.

## Final Check

Before returning a draft, confirm that the text makes clear:

- what was observed
- in what direction the result goes
- what interpretation is justified
- what interpretation is not yet justified
